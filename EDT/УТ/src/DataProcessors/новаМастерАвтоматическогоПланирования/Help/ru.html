Обработка "Автоматическое планирование доставки" (Решение транспортной задачи)

Обработка предназначена для автоматического распределения заказов на доставку по доступному транспорту (формирования плана доставки) с использованием настраиваемых показателей целевой функции. В результате работы алгоритма будут созданы документы "Рейс" с соответствующим составом для которого суммарные показатели целевой функции будут наилучшими.

Закладка "Общие"

  • Реквизит "Алгоритм" - выбор используемого алгоритма автоматического планирования из доступных.
  • Реквизит "Максимальное использование транспорта" - при установке данного флага алгоритм будет в первую очередь, пытаться создавать вторые и третьи рейсы в день для уже задействованного транспорта иначе вторые и третьи рейсы будут участвовать в генетическом отборе наравне с загрузкой нового транспорта.
  • Реквизит "Коэффициент времени разгрузки" - при указании коэффициента отличного от 0, на данный коэффициент будут умножено время разгрузки во всех доставках одновременно. Данная функция может применяться, что бы увеличить время разгрузки во всех точках одновременно без изменения информации в заказах, например для доставки в предпраздничные дни.

Закладка "Целевая функция"

На данной закладке указываются весовые параметры показателей целевой функции.

  • Длительность - минимальная длительность рейсов по времени
  • Расстояние - минимальный пробег
  • Количество точек - максимальное количество обслуживаемых клиентов в одном рейсе
  • Вес - максимальная загрузка кузова по весу
  • Объем - максимальная загрузка кузова по объему
  • Разброс точек - минимальный разброс между точками в рейсе (маятниковые рейсы)
  • Расстояние без склада - минимальный пробег без учета расстояния от склада до первой точки разгрузки и от последней точки разгрузки до склада
  • Глубина оптимизации - определяется глубина поиска решения. Чем больше этот параметр тем больше времени тратится на поиск. Оптимальное значение = 10.
  • Размер начальной популяции - определяет размер начальной выборки для поиска решения. Оптимальное значение = 20.

Закладка "Качество рейсов"

На данной закладке указываются минимально допустимое качество сформированных рейсов. Если в результате работы алгоритма не удается сформировать рейсы удовлетворяющие этим качествам, то остаются нераспределенные заказы.
