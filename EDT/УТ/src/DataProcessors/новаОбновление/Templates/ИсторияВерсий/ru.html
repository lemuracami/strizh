Редакция 3.5

Редакция 3.5 конфигурации "Управление транспортной логистикой" для работы требует установки сервера "Корпоративнй сервер логистики".

Серверная часть (Устанавливается на компьютер, который будет выполнять роль сервера)
1. .Net Framework 4 – основной компонент необходимый для работы корпоративного сервера логистики. Обязательно надо устанавливать первым.
2. HASP SRM – драйверы системы защиты HASP. Необходимо устанавливать да же, если используется версия без ключа.
3. PostgreSQL – сервер баз данных
4. Корпоративный сервер логистики

Клиентская часть (Устанавливается на каждый компьютер, на котором будет использоваться конфигурация «Управление транспортной логистикой ред . 3.5»)
1. .Net Framework 4 – основной компонент необходимый для работы корпоративного сервера логистики. Обязательно надо устанавливать первым.
2. Клиент корпоративного сервера логистики

Переход с версии 3.2

Для перехода с версии 3.2 конфигурации "Управление транспортной логистикой" необходимо:

  1. Установить серверную и клиентскую часть "Корпоративного сервера логистики"
  2. Создать базу в консоли администрирования корпоративного сервера логистики
  3. Объединить конфигурации
  4. Запустить обработку "Переход на версию 3.5"
